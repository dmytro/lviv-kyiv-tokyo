<p>

<font face="Arial,Helvetica">The font sizes and margins of these files are smaller than
in the documentation shipped with the PowerPack to allow for fewer 
printed pages. Make sure your printer is properly setup to avoid missing
some of the contents.</font>
<p><font face="Arial,Helvetica">Three paper sizes are available: <a href="#a4pdf">A4</a>, <a href="#legalpdf">US legal</a> and <a href="#uspdf">US letter</a>. </font>
<p><font face="Arial,Helvetica">
 Files are in  compressed formats to allow for quicker  downloads. The final uncompressed size is also provided to notify you of the storage space required on a hard drive.  Files are compressed in two formats:

<ul>
<li> <b>.tar.gz</b> (gzip; default on all GNU/Linux distributions) Works with most of the decompression utilities on many platforms (Winzip, Stuffit, etc.)</li>
<li> <b>.tar.bz2</b> (<a href="http://sourceware.cygnus.com/bzip2/index.html">bzip2</a>) Provides higher compression but doesn't work with as many  decompression utilities.</li>
</ul>

<p><font face="Arial,Helvetica">NOTE:</font>
<ul>
<li>
<font face="Arial,Helvetica">The original file formats are PS. PDF was produced
from the PS versions and the conversion process may have introduced modifications.</font></li>

<li>
<font face="Arial,Helvetica">If you cannot view PDF with your system or software, be warned that the PS
versions are quite large (check file sizes before downloading).</font></li>

<li>
<font face="Arial,Helvetica">The page numbers and chapter titles may
be missing in some PDF versions.</font></li>

<li>
<font face="Arial,Helvetica">On GNU/Linux, gv may not  correctly
display some pages of the PS versions. This is a bug of gv,
gs and lpr do work correctly.</font></li>
</ul>
